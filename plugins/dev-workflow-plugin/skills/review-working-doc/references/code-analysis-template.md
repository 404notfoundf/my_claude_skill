# 局部代码分析模板

适用于当前项目中的单文件分析、局部实现审查、函数/模块行为说明、风险点与改进点并存的技术分析文档。

## 推荐骨架

```md
# 文档标题

一句话说明分析目标，例如：分析某个文件、函数或局部模块的实现质量与风险。

说明：

- 本文档聚焦局部代码分析，不等同于安全漏洞台账
- 既可以记录问题，也可以记录实现特点、边界行为和优化建议

## 总览

| ID | 类型 | 标题 | 优先级 | 处理建议 | 备注 |
|---|---|---|---|---|---|
| 1 | 风险 | 鉴权条件缺失 | 高 | 建议修复 |  |
| 2 | 优化 | 状态字段重复维护 | 中 | 建议优化 |  |
| 3 | 说明 | 当前实现依赖缓存预热 | 低 | 保持现状 | 设计约束 |

## 详情

### 1. 鉴权条件缺失

- 类型：风险
- 优先级：高

**现状分析**

- 说明当前代码怎么实现
- 说明触发条件或上下文

**影响**

- 说明这会导致什么问题、维护成本或行为风险

**建议**

- 说明推荐改法
- 如果暂时不改，也要说明保持现状的前提

**相关位置**

- `path/to/file.go:12-34`

## 建议处理顺序

1. 先处理高优先级风险项
2. 再处理影响维护成本的优化项
3. 最后保留说明类条目作为上下文记录
```
