# 问题 / 整改台账模板

适用于当前项目中的安全审查、代码问题、权限缺陷、行为错误、信息泄露、整改跟踪类文档。

## 推荐骨架

```md
# 文档标题

一句话说明这份文档的目的。

说明：

- 这份文档基于什么来源得出
- checkbox 如何维护
- 总览表是唯一状态来源

## 总览

| ID | 级别 | 问题 | 修复 | 风险类型 | 备注 |
|---|---|---|---|---|---|
| 1 | P0 | 问题标题 | [ ] | 信息泄露 | 待排期 |
| 2 | P1 | 问题标题 | [x] | 权限绕过 | 已完成 |

## 详情

### 1. 问题标题

- 级别：P0

**风险说明**

- 说明为什么这是问题
- 说明影响面和后果

**修复建议**

- 说明推荐改法
- 如果有分步修复，可以列 2-3 点

**代码位置**

- `path/to/file.go:12-34`

## 推荐实施顺序

1. 先处理 P0
2. 再处理 P1
3. 最后处理 P2

## 验收建议

- 补什么测试
- 核查什么行为
- 哪些边界场景要复验
```
